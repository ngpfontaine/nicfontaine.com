Zoom Scheduler

¶

A small session-scheduling webapp created to connect a group of learners, over a few-week timespan, to multiple Zoom accounts for simulation delivery. Built on NodeJS with Express to manage views, and using the ZOOM RestAPI to create and cancel meetings. Without the need for a full database, sign-up contact info & zoom url are entered into a JSON file organized by date.

Nodemailer is used to send booking confirmation emails, and are set to send a reminder before the scheduled session time. Booking emails contain cancellation links that clear the Zoom meeting entry, reminder-email flag, and booking slot.

It's quick, light, and easy to re-skin for new clients' pilot deliveries.

Start Page

¶

I wanted a personalized start page with common links I use, search features, and relevant hud stats and naturally it had to be navigable w/o the use of a mouse. Additionally, I wanted local info like weather, time, and IP auto-configured; and important information legible from across the room w/o taking over the smaller elements in the viewport.

You can randomly change the colour palette, change search sites, and navigate the link grid all using the keyboard. The tile grid of links is calculated by vertical offset, and re-logs on browser resize so it should handle any number of additional or subtracted link tiles.

Background weather images are loaded in response from local weather data polled every 20s, and crossfade when weather conditions change. Mini search icons are listed below the search input to correspond with search sites; and switch when clicked, L or R arrow keys are used, or the search field is swiped on mobile.

I've included arrays to easily add more search sites, respective placeholder names for them, and corresponding icons. I regularly add new features, and you're welcome to check out and/or fork on github. I'm glad to help out if you're not completely familiar with js/html/css and want to personalize your own.

Gravity Forms Validation

¶

Unhappy with form validation on our landing pages at Element-360 I created an extension with JQuery, exclusively to extend the capabilities of Gravity Forms. In watching behaviour analysis recordings I found a large number of people clicking the CTA to immediately receive information. This results in a page reload with error messages appended to the required fields - not an optimal user experience.

My goals were:

  • Live validation/disable button send until form fulfilled
  • UI change from button click - of blank required fields
  • Validation check (logic and visual) on field blur
  • Email regex validation
  • Client-side validation, w/o AJAX
  • Simple, safisfying UX (people have a mote of tolerance on landing pages)

Additionally I implemented a button text update on send - as on slower networks, lag is generally perceived as a site bug and users will spam-click out of frustration. An easy way to prevent form submissions, while adding a useful onclick event, was to place a 'cover' button over the gravity forms send button. Clicks on the cover check required fields and add classes to notify the user. Field blur, focus, and change events check input validation and form completion, while adding and removing visual classes.
